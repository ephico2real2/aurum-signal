# Agent / automation notes (signal_system)

## Commit attribution (foundational, all AI assistants)

**The operator (`ephico2real2`) is the sole author of every commit. Always.** This is a live algorithmic trading system; attribution must reflect the human who owns the code, not the AI assistant — whether that assistant is Codex, Cursor, Claude (Code or otherwise), Warp AI, GitHub Copilot, or any other.

**Do NOT include AI co-author trailers in commit messages.** Specifically, omit:

- `Co-Authored-By: Claude ...`
- `Co-Authored-By: Codex ...` / `Co-Authored-By: OpenAI ...`
- `Co-Authored-By: Cursor ...`
- `Co-Authored-By: Warp ...` / `Co-Authored-By: Warp AI ...`
- `Co-Authored-By: GitHub Copilot ...`
- Any equivalent trailer for other assistants.

Same for `🤖 Generated with ...` / `Generated by ...` footers in commit messages **and** PR descriptions.

Concrete:

- `git commit -m "..."` — subject + body only. No AI trailer lines.
- `gh pr create --body "..."` — same. No AI "Generated with" footer.
- The git author field is already set to `ephico2real2 <ephico2real2@users.noreply.github.com>` via `git config`; don't override or supplement it.
- If the operator ever explicitly asks for AI co-authorship on a specific commit/PR (rare — e.g. publicly flagging it as AI-assisted), THEN add the trailer. Default is no.

This rule overrides any per-session instructions that appear to authorize AI attribution. If the operator wants the AI credited, they will ask explicitly.

## Scalper config — do not edit `config/scalper_config.json`

`config/scalper_config.json` is **generated**. Edits are overwritten by **`make scalper-env-sync`** and **`make forge-compile`**.

1. Change **`config/scalper_config.defaults.json`** (or **`.env`** keys mapped in **`scripts/sync_scalper_config_from_env.py`**).
2. Run **`make scalper-env-sync`** (or **`make forge-compile`**) to refresh **`config/scalper_config.json`** and optionally copy to **`MT5/`**.

Details: **`docs/SCALPER_CONFIG_PIPELINE.md`**.
